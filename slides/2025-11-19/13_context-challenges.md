## 挑战 1：时间跨度

**场景**：今天用 Claude Code 写了一半功能，明天继续

**问题**：
- AI 不记得昨天的对话
- 你可能忘记当时的设计思路
- 代码改了，但为什么改的理由丢失了

**现实案例**：
```
今天：实现了用户登录功能，但密码加密方式还没决定
明天：打开 Claude Code → "帮我继续做登录"
AI: "好的，需要什么登录功能？" ❌ (上下文丢失)
```

## 挑战 2：环境差异

**场景**：在公司电脑开发，回家想继续

**问题**：
- 不同机器上的配置不同
- Git 提交了代码，但"为什么这样写"没提交
- AI Agent 的会话历史不同步

**现实案例**：
```
公司：用 Cursor 实现了复杂的数据处理逻辑
家里：打开 Cursor，看到代码，但不记得当时的权衡考虑
     为什么用 A 方案而不是 B 方案？
```

---

## 挑战 3：工具切换

**场景**：从 Cursor 切换到 Claude Code CLI

**问题**：
- 不同 AI 的"记忆"不互通
- 之前的对话内容无法迁移
- 需要重新解释项目背景

**现实案例**：
```
Cursor: 帮我实现 A 功能 (讨论了 30 分钟，确定了方案)
Claude Code: 帮我继续实现 A 功能
AI: "好的，你想怎么实现？" ❌ (需要重新讲一遍)
```

---

## 挑战 4：团队协作

**场景**：A 同事写了一半，B 同事接手继续

**问题**：
- A 的思路和决策没有记录
- B 不知道 A 用 AI 时讨论了什么
- Code Review 只看代码，不知道背后的 reasoning

**现实案例**：
```
A: (用 Claude Code) 实现了一个算法，经过 3 个迭代版本
B: 接手代码，看到最终版本，但不知道：
   - 为什么放弃了前两个版本？
   - 有哪些已知的边界情况？
   - 还有什么技术债务？
```
