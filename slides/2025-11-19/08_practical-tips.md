## 实用判断技巧

### 如何判断 Code vs Data？

**问自己**：
- 这个东西会被执行/解释吗？ → **Code**
- 这个东西被程序处理吗？ → **Data**

**安全原则**：永远不要把不可信的 Data 当作 Code 执行！
- ❌ `eval(user_input)`
- ❌ `exec(user_input)`
- ✅ 使用参数化查询、白名单验证

## 实用判断技巧

### 如何判断 Configuration vs Data？

**问自己**：
- 这个值控制程序**怎么运行**吗？ → **Configuration**
- 这个值是程序处理的**内容**吗？ → **Data**

**实践建议**：
- Configuration 放在环境变量、配置文件
- Data 放在数据库、用户输入
- 敏感 Configuration (API keys) 要加密

## 实用判断技巧

### 如何设计 Stateless 函数？

**三个原则**：
1. **只依赖输入参数** (不读取全局变量、文件、数据库)
2. **不修改外部状态** (不写文件、不改全局变量)
3. **相同输入 → 相同输出** (可预测)

**好处**：
- ✅ 容易测试 (不需要 mock 外部状态)
- ✅ 容易并行化 (无竞争条件)
- ✅ 容易理解 (逻辑自包含)

## 关键要点

1. **没有绝对边界** - 这些概念是工具，不是真理
2. **Context Matters** - 取决于你在讨论什么层面
3. **实用主义** - 问"这样区分对我有帮助吗？"
4. **团队共识** - 和团队统一术语理解
