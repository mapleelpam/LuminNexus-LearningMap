# LearningMap 網站 UI 設計提案

## 設計概念

**品牌軸**：domain 是 `prisma.vision` → 稜鏡（prism）→ 白光折射成光譜。
把「光譜漸層」當品牌識別元素，節制使用（logo、hero 底線、進度條），
底色維持暖白紙感，符合「學習教材」的閱讀氛圍。

**三個核心畫面**：

**首頁資訊架構 = 三層消費模式**：
① general（共同起點，循序）→ ② roles（分流，循序）→ ③ 參考資料庫（查閱型：tools / data-sources / projects）。
前兩層是「學習路徑」，第三層是「查資料」，版型刻意不同（卡片 vs 清單庫）。

**site 收錄範圍**：general、roles、tools、data-sources、projects 全上站；
archive 排除；slides（Marp 分頁檔、無 frontmatter）不當一般文章收錄——
可另用 marp-cli build 成 HTML 簡報後以連結掛在站上（開放決策）。
- Hero：一句話說清楚「新人 3-5 天上手」
- 主體：四張角色卡（Testing & BA / UI-UX / Crawler / PM），
  各標示天數、主題數、適合對象 → 點擊直達該角色學習路徑
- 次要區：通用核心能力（general/）入口，理論系列與實務層用 chip 呈現
- 頂欄：站內搜尋（中文分詞）

### 1a. general 區塊方案（三選一，prototype 可即時切換）

**A 分軌清單**（sitemap 式）：一個大框、四欄直式連結，結構一目瞭然、資訊密度最高
```
┌ 📚 General Learning Map ──────────────── 完整大綱 → ┐
│ 🧱基礎核心    🤖AI 協作      💭理論系列    🛠️實務層  │
│ Unix/Linux   Claude Code    湧現與D/C    許願與計畫  │
│ 資料工程      ContextOps    張力與價值    四象限     │
│              知識管理       同構與投影    Agent形態  │
└────────────────────────────────────────────────────┘
```

**B 分軌卡片**（與角色卡同一視覺語言，小一號）：首頁整體一致性最好，但看不到各篇標題
```
┌🧱 基礎核心 ┐ ┌🤖 AI 協作 ┐ ┌💭 理論系列 ┐ ┌🛠️ 實務層 ┐
│一句話描述… │ │一句話描述…│ │一句話描述… │ │一句話描述…│
│[2 篇]     │ │[4 篇]     │ │[6 篇]      │ │[3 篇]    │
└───────────┘ └───────────┘ └────────────┘ └──────────┘
```

**C 階段地圖**（橫向流）：帶「建議由左往右」的學習順序感，呼應「地圖」的隱喻

- 左：該角色的主題樹（收合其他角色）
- 中：文章內容（Mermaid 圖原生渲染）
- 右：本頁目錄（TOC），滾動高亮
- 底部：上一篇/下一篇（照學習順序）
- 這一層 = MkDocs Material 原生版型 + 客製 CSS，成本最低

## 與 MkDocs Material 的落地對應

| 畫面 | 實作方式 | 客製成本 |
|------|---------|---------|
| 首頁角色分流 | Material custom homepage（override template） | 中（一次性） |
| 路徑頁時間軸 | 各角色 00_outline 改造 or 附加 CSS class | 低-中 |
| 閱讀三欄 | Material 原生 | 近零 |
| 進度勾選 | 少量 custom JS + localStorage | 低 |
| 光譜品牌色 | CSS variables override | 低 |

## 開放的設計決策

1. **主色**：prototype 提供三組主色切換（靛藍/磚橘/森綠）+ 光譜漸層固定為輔助識別
2. **進度追蹤**：要不要「已讀」勾選？（本機 localStorage，無帳號、無後端）
3. **首頁調性**：目前走「溫暖紙感」，如果想要更「科技產品感」可改冷色系
