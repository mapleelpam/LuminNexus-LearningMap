01問題 · 兩種語言不可比較

今天，這兩句話活在不同的宇宙

health-claim · KOL/科學

「Ashwagandha 顯著降低皮質醇、改善壓力反應。」

ashwagandha─[efficacy,pro]▶stress
強、可證偽、帶 evidence（RCT）
benefit-claim · PDP

「含 KSM-66® 南非醉茄，支持每日的平靜與放鬆。」

ashwagandha─[composition]▶"calm"
弱、hedged、法規閹割（structure-function）

同一顆成分、同一個訴求方向，但「說法」的型別不同——一個在講因果、一個只敢講成分。 人眼看得出「不對勁」，但系統看不出來，因為兩者的 schema 不同、不可運算。要找張力，得先讓它們落進同一個空間。

Σ · 共同表示空間
S─[P,valence,evidence]▶O

「保結構」有三條硬條件，缺一則兩邊落進 Σ 也對不齊：

S 對到同一 canonical entity space——ashwagandha 就是 ashwagandha，不管來自 reel 還是產品頁。
O 對到同一 canonical outcome space——KOL 的 stress 要能對上 PDP 的 "calm"。（見 04）
P 對到同一 predicate enum——efficacy / mechanism / composition… 是共用的一組維度。（見 05）

紅利：一旦三條成立，跨 domain 比較從「不可通約的兩種語言」變成「同一空間的向量運算」。不必造第二套分析工具。

04O 對齊 · 假同構與影子

「語氣像」不是「講同一件事」

embedding 是把句子投影到「語義相似度」空間的一個 projection。任何 projection 都有 kernel（被壓成零、不可回復的那些維度）—— 而它壓掉的，恰好是我們要的指稱。cosine 在影子空間的近，拉不回物件空間的同一。

同樣是投影，差別在 kernel 裡裝了什麼：Σ 投影保留指稱（這是真同構的目標，見 02）； embedding 投影壓掉指稱——把「影子相似 ⇒ 物件同一」當成保結構映射，就是假同構。
假同構 · 高 cosine，錯
weight-lossweight-gain ──▶≈ 同一向量
不同指稱、同一影子（投影非單射）。共享 "weight" 就被判相似——語氣像 ⇏ 同一件事。
漏抓 · 低 cosine，也錯
stress-relief──▶ "promotes calm"reduce cortisol
同一指稱、兩個散影子。跨語域（行銷 vs 臨床）／跨語言，向量離很開——講同件事 ⇏ 語氣像。
embedding 的 kernel（壓成零）指稱 · 方向 · valence
embedding 的主軸（保留）語氣 · 語域 · 字面

把影子當物件：任何單一報告都是投影。

忘了這件事的人，會拿自己的影子跟別人的影子吵「誰才是真的」——兩個都是影子。工程結論：存物件，不存影子——存 S-P-O／力（見 06），不存 embedding 向量或某個視角的報告。判定靠 LLM 讀 proposition（讀物件本身）；embedding 只做 blocking（便宜預聚群）。

stress_reduction"promotes calm""支持放鬆" ──vocab-judge──▶ stress-relief

05力 = 向量 · predicate 是 facet 不是 scale

一次看 1–2 個 predicate，
不塌成一個「強度」純量

predicate enum 的每個成員本身就是一個力的維度，彼此不可通約、也不該通約。 強度序（lattice）會把多維的力壓成一個數——那恰恰摧毀了「價值多元」。 選哪個 predicate = 選一個 lens。這就是 MDFO：predicate 是一個可 filter 的 Dimension。

efficacymechanismsafety compositionassociationcomparison
lens（選定 predicate）health 端PDP 端讀出的力
efficacypro（有 RCT）— 缺席coverage gap ▸ 品牌沒跟上背書
efficacyconpro正面對撞 ▸ 監管紅旗
composition— 缺席pro（含 X mg）裸宣稱 ▸ 有成分無實證
safetycon（劑量警告）— 缺席風險落差 ▸ 揭露缺口
efficacypropro（s-f 措辭）一致 ▸ 可信定位

力是向量：分量 index by〈predicate × valence × 在場 × evidence〉。永不塌成純量——每個 lens 顯示一種不同的力。

06兩層 · 力（客觀單一）／張力（主觀多元）

力存在資料層，張力發生在視角層

力 · Force

客觀 · 單一 · 收斂

Σ 圖上的 typed edge。對一組 (S,O) 對齊的宣稱，力是各 lens 分量組成的向量——市場的「truth」，收斂成一張圖。

F(h, b) = ⟨ per-predicate: Δvalence, Δ在場, Δevidence ⟩

張力 · Tension

主觀 · 多元 · 不收斂

某個 position 拿自己的權重去投影那個力向量。同一張力圖，不同角色看見不同張力——市場的「value」，永不收斂成一個。

Tposition = wposition · F
監管端 ▸ 重 Δpredicate + Δevidence（越界/裸奔） 品牌商 ▸ 重 coverage gap（錯過的定位） 原料商 ▸ 重 S 層 aggregate（聲量 vs 採用） 學術端 ▸ 重 Δevidence 全貌（實證縫隙）

這就是 tension-value-perspective §2「事實單一、價值多元」的工程化：存力，不存張力。而每個 w 都自帶一個盲區（null space）——凡與 w 正交的分量，內積後歸零，你不加權的東西結構性地看不見（見 08）。

07partial 也是力 · negative space

只共享 S、或只共享 O，
就是機會地圖

力不需要 S 和 O 都對齊。partial match 是不同維度的力——空缺（negative space）往往比對撞更值錢：

full match

S ✓ O ✓ → 直接比較（對撞／一致／落差）

最尖銳

只共享 S

同成分、不同訴求 → 這顆成分在 health 端被賦予哪些 O、PDP 端只賣哪些 O

原料商：訴求缺口

只共享 O

同訴求、不同成分 → 誰在爭奪「helps sleep」這個位置

品牌商：定位競逐

每種 partial 是一個獨立維度的力，不混。要不要分級（先跑 full、partial 當第二階）是個工程取捨——見待拍板。

棋盤：為什麼是平台

每個角色的 w 都自帶一整個 null space——凡與 w 正交的分量，內積後歸零、結構性看不見。唯一不做投影、直接持有完整 F 的，就是平台。

「看見全局」不是口號，是代數：投影必失維（見 04），不投影才保全。§4「看得見真實張力的人，控制遊戲。」

我們賣的不是資料，是棋感——把市場的力攤開給看得懂的人。

別人手上是投影後的影子（少了維度、看不見自己的盲區）；平台手上是那個向量本身。

09待拍板 · 這幾點會定 schema

下一步：schema 先行，還是資料先行？

  • A · O canonical map 的建法走既有 vocab workflow（cluster→propose→reconcile）套 object slot；embedding 僅 blocking。要不要為 PDP 的 benefit-language 開一個 O 專屬的 alias family？
  • B · partial 力要不要分級full match 先跑，只共享 S / 只共享 O 當第二階？還是一開始就全圖（會爆量）。
  • C · predicate enum 要補 structure-functionPDP 主力是 s-f 措辭，現有 enum（efficacy/mechanism/composition…）還沒這個成員。補一個，還是把它併進 composition + 弱 valence？
  • D · 先驗一個假設抓一小批 PDP 投影進 Σ，看 predicate 分佈——若大量落在 s-f/composition（像 LinkedIn 品牌頁 90% discourse），那力的主體是「health-efficacy ↔ PDP-composition 的系統性錯配」，跟 Fugue 的「兩種沉默」同源。

我的建議：先做 D（一小批 PDP 實測，一天內），用真實 predicate 分佈決定 A/C 的形狀，再定 schema。
